# Rewrite Backend: Laravel → Express (TypeScript + Prisma + PostgreSQL)

## Context

Project ini adalah skripsi POS Restaurant (Ezra Brilliant, C14220315) untuk "Ayam Bakar Banjar Monosuko". Naskah skripsi terbaru telah diubah ke stack **all-JavaScript**, tapi codebase masih Laravel + MySQL. Project harus disinkronkan dengan naskah.

**Masalah riil di resto yang ingin dipecahkan (dari cerita user + foto buku):**
1. Semua order + stok masih dicatat manual di buku kertas (kiri: stok, kanan: penjualan). Buku kadang tidak lengkap — pegawai lupa opname pagi → kasir tidak tahu stok.
2. Kasir kadang langsung iya-kan order tanpa cek stok → pernah harus GoSend dari rumah pemilik ke resto untuk kirim stok yang habis.
3. Pemilik tidak tahu persis pengeluaran/pemasukan bulanan — hanya estimasi.
4. Pegawai kasir = anggota keluarga pemilik → bukan tech-savvy → UX harus sangat simpel.

**Pendekatan pembangunan:** User ingin **incremental, pedagogical** — dibangun 1 file (atau 1 grup kecil yang saling terikat) per step. Tiap step dijelaskan fungsinya & cara nyambungnya, lalu menunggu review sebelum lanjut. Tujuan user: *paham* prosesnya, bukan sekedar dapat hasil jadi.

---

## Decisions (sudah dikonfirmasi user)

| # | Keputusan | Detail |
|---|---|---|
| 1 | **Stack backend** | Node.js 20 LTS + Express 4 + TypeScript + Prisma ORM + PostgreSQL 16 (self-host) + JWT + Zod + Vitest + Pino |
| 2 | **Struktur folder** | Replace total: `backend/` Laravel dihapus, diisi Express. Folder lama direname dulu ke `backend.laravel.bak/` sebagai safety net, dihapus setelah user konfirmasi semuanya jalan. |
| 3 | **Data existing** | Migrate semua: users + menus + daily_menu_stocks + transactions + transaction_items + settlements. Export JSON dari MySQL, import ke Postgres via script. |
| 4 | **API contract** | 1:1 dengan Laravel lama. Base path `/api`, port `8000`, response shape `{success, message, data}`. Frontend tidak diubah (kecuali auth header JWT vs Sanctum — sama-sama `Authorization: Bearer <token>`, transparan). |
| 5 | **Roles** | 3 roles: `owner`, `cashier`, **`kitchen`** (baru). Kitchen tugas utamanya hanya input opname pagi — tidak ada notifikasi order, tidak ada workflow kompleks (menghindari scope creep + risiko adopsi). |
| 6 | **Payment methods** | Enum detail: `cash`, `qris`, `transfer` (manual transfer), `debit`, `credit`, `ojol` (Gojek/Grab/ShopeeFood). Settlement split mengikuti enum ini. |
| 7 | **Bahasa code** | Full English — nama variable, kolom DB, enum, komentar. Update `CLAUDE.md` + `AGENTS.md`. |
| 8 | **PIN storage** | Plaintext `VARCHAR(6)` dengan index (sama seperti Laravel — agar bisa lookup cepat saat login). Security trade-off didokumentasikan di `SKRIPSI.md`. |
| 9 | **Diskon** | Kolom `discount_amount` dipertahankan (optional usage). |
| 10 | **PWA** | Level A — installable (add-to-home-screen) + static asset cache via `vite-plugin-pwa`. TETAP butuh internet untuk data. Alasan: kasir selalu pakai wifi resto, user bukan tech-savvy, hindari risiko sync failure. |
| 11 | **Owner-side (expenses + dashboard)** | **Deferred ke akhir**. Fokus restaurant-side dulu (POS, tables, menu, stock, settlement). Owner-side jadi Phase 13. |
| 12 | **Tracking** | Buat `ROADMAP.md` untuk track completion per step. Buat `SKRIPSI.md` untuk catat perubahan naskah skripsi. |
